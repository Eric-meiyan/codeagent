The fastest way to make an online coding agent useful is to preserve the tool it already understands: a terminal. hicode starts by rendering a remote PTY through xterm.js and keeping the durable unit inside tmux.

That approach makes the MVP practical. The agent can run inside a cloud sandbox, use real project files, call a managed model gateway, and expose previews without first rebuilding every interaction as a custom chat protocol.

Long term, the product can add structured events for file diffs, command results, approvals, token accounting, and audit trails. The terminal path gets users to a working development loop first; structured control can grow around it.
